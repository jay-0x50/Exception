# 보스 콘셉트 시트

<aside>

**목적:** `Exception`의 보스들이 어떤 프로그래밍 언어/개발 도구에서 출발했는지, 그리고 이를 액션 보스전으로 어떻게 바꿀지 정리한다.

</aside>

## 1. 전체 콘셉트

플레이어는 오류가 폭주한 런타임 내부로 진입하고, 각 보스는 특정 언어/도구의 상징이 변형된 예외 개체로 등장한다.

- 보스는 단순 패러디가 아니라, 각 언어/도구의 이미지와 개발자 밈을 전투 패턴으로 번역한다.
- MVP에서는 보스 1체만 완성하고, 나머지는 확장 후보로 보관한다.
- 포트폴리오에서는 “콘셉트가 시스템으로 연결된다”는 점을 보여주는 것이 핵심이다.

| 후보 | 모티브 | 실루엣 | 전투 키워드 | 구현 우선순위 |
| --- | --- | --- | --- | --- |
| Python Boss | Python 언어 | 뱀 | 유연한 궤도, 감기, 독 장판, 예측 꺾기 | 1순위 |
| PHP Boss | PHP 언어 | 코끼리 | 돌진, 충격파, 묵직한 전방 압박 | 확장 |
| MySQL Boss | MySQL | 돌고래 | 물결, 회전, 데이터 파편, 범위 장악 | 확장 |
| Perl Boss | Perl | 낙타 | 변칙 이동, 문자열/실타래, 불규칙 패턴 | 확장 |

## 3. MVP 보스 제안: Python Boss

### 핵심 이미지

- 긴 몸체와 꼬리로 공격 범위를 크게 보여줄 수 있음
- 구불구불한 이동 궤적으로 Phase 2의 압박을 만들기 좋음
- 독/코드 조각/들여쓰기 라인 같은 시각 요소를 패턴에 섞기 쉬움

### Phase 1

| 패턴 | 역할 | 대응 의도 |
| --- | --- | --- |
| Tail Sweep | 근거리 기본 압박 | 텔을 보고 회피하거나 Normal 판정이면 패링 |
| Coil Slam | 중거리 광역기 | 크게 피한 뒤 후딜에 반격 |

### Phase 2

| 패턴 | 역할 | 대응 의도 |
| --- | --- | --- |
| Shadow Dash | 원거리 접근기 | 측면 회피 후 보스가 지나친 틈에 반격 |
| Syntax Venom | 장판/공간 제어 | 안전 지대로 이동하며 락온 유지 |

## 5. 비주얼 가이드

- 과하게 귀엽게 만들기보다, 원본 아이콘의 특징을 “보스 실루엣”으로만 남긴다.
- 색은 언어/도구의 대표색을 참고하되, 전투 가독성을 우선한다.
- 텔 동작은 동물적 움직임으로 읽히게 만들고, 실제 판정은 명확한 기하 형태로 관리한다.

## 6. 구현 메모

- 보스 콘셉트는 BP/애니메이션 레이어에서 바뀌어도 C++ 코어는 재사용 가능해야 한다.
- `BossSkillData`에 `SkillTag`, `AttackType`, `DamageProfileId`, `MontageKey`를 넣어 보스별 패턴 교체가 가능하게 설계한다.
- MVP 이후 다른 언어 보스를 추가할 때는 같은 `AExBossCharacter`와 `UExBossSkillComponent` 구조를 유지한다.
