# Exception — 1편 전체 스토리 아크

---

## 등장인물

| 이름 | 정체 | 역할 |
|---|---|---|
| **Handel** | The Runtime이 인스턴스화한 처리자 | 주인공 |
| **Nel** | The Runtime의 마지막 잔재 / 자신의 정체를 모름 | 보조 캐릭터 (멜리나 포지션) |
| **Terminal** | 자신이 도구였음을 알아버린 선대 처리자 | 최종 배후 악역 |
| **Vethara** | Python 최초의 RecursionError에서 탄생한 Primordial Exception | 보스 1 |
| **Aurathos** | Python 최초의 IndentationError에서 탄생한 Primordial Exception | 보스 2 (Vethara와 듀얼) |
| **Fatal Elephant** | PHP 최초의 Fatal Error에서 탄생한 Primordial Exception | 보스 3 |

---

## 세계관 키워드

| 현실 | The Runtime 내 의미 |
|---|---|
| 프로그래밍 언어 | 신앙 / 종교 |
| 오류 (Exception) | 봉인된 신격 존재 |
| 디버깅 | 봉인 의식 |
| Handler | 예외를 처리하도록 설계된 프로세스 |
| Terminal | 모든 프로세스가 종료되는 곳. 끝. |
| Null | 정의되지 않은 것. 존재하지만 값이 없는 것. |

---

## 전체 Act 구조

```
Act 1 — 인스턴스화
Act 2 — 첫 번째 예외 (Serpent.py)
Act 3 — 균열
Act 4 — 두 번째 예외 (Fatal Elephant)
Act 5 — 진실의 문 앞에서
Ending A — 정의 (일반 엔딩)
Ending B — 비극 (히든 엔딩)
```

---

## Act 1 — 인스턴스화

### 줄거리

The Runtime 어딘가에서 Handel이 갑자기 깨어난다.
기억도 없고, 자신이 무엇인지도 모른다.
내장된 명령어 하나만 있다.

```
TASK: Terminate all unhandled exceptions.
```

혼란 속에서 Nel이 나타난다.

> *"처음이지? 놀랍지 않아. 네가 막 인스턴스화됐으니까."*
> *"나는 Nel. 어디서 왔는지는 나도 몰라. 그냥 여기 있어."*
> *"일단 움직여. 이 세계가 무너지고 있거든."*

Nel은 Handel에게 The Runtime의 현재 상황을 설명한다.
봉인이 풀리고 있다. Primordial Exception들이 깨어나고 있다.
누군가 의도적으로 봉인을 건드리고 있다.

Handel은 아무것도 모르지만 내장된 명령대로 움직이기로 한다.

### Nel의 첫 번째 부탁 (히든 루트 트리거)
> *"출발하기 전에 하나만. 저기 저 데이터 잔재들 있지?*
> *오래된 것들이야. 나는 저게 뭔지 알고 싶어. 네가 수집해줄 수 있어?"*

- 일반 루트: 무시하고 진행 가능
- 히든 루트: 수집 시 **"이전 프로세스의 잔재"** 로그 파편 획득

---

## Act 2 — 첫 번째 예외 (Serpent.py)

### 줄거리

Handel과 Nel이 첫 번째 봉인 구역에 도달한다.
Python 영역. 바닥에 들여쓰기 라인이 새겨진 원형 공간.

봉인이 이미 절반쯤 풀려 있다.
두 개의 거대한 실루엣이 공간을 가득 채우고 있다.

**Vethara** — 끝없이 자신을 호출하며 멈추는 법을 잃어버린 존재.
**Aurathos** — 들여쓰기 하나로 세계가 무너진다는 것을 아는 존재.

Nel이 조용히 말한다.
> *"저들은 원래 이 세계의 법칙에서 태어난 거야.*
> *오류가 없었다면 수정도 없었겠지. 저들도 그냥 존재했던 거야."*

Handel은 명령대로 처치한다.

### 보스전 후

Aurathos의 마지막 말.
> *"왜 너는 우리를 지우려 하는가.*
> *우리가 없었다면 너도 없었다."*

Vethara의 마지막 말.
> *"...너도 결국 반복될 것이다."*

처치 후 봉인 구역에서 **Terminal의 흔적**이 발견된다.
누군가 이곳에 먼저 있었다. 봉인을 건드린 흔적.
하지만 아직 Terminal이 누구인지는 알 수 없다.

### Nel의 두 번째 부탁 (히든 루트 트리거)
> *"저 처치된 자리에 뭔가 남아있어.*
> *보통은 그냥 사라지는데... 이건 달라. 가져올 수 있어?"*

- 일반 루트: 무시하고 진행 가능
- 히든 루트: **Vethara / Aurathos의 마지막 로그 파편** 획득
  - 파편 내용: *"우리를 깨운 자는 우리와 같은 존재였다. 처리자."*

---

## Act 3 — 균열

### 줄거리

첫 번째 봉인 구역을 지나 더 깊은 곳으로 이동하는 Handel과 Nel.

이동 중 Nel이 이상한 것들을 발견하기 시작한다.
오래된 데이터 잔재들. 이전에 누군가 여기를 지나간 흔적들.
Handel의 것과 구조가 거의 동일한 흔적.

Nel은 말하지 않는다. 혼자 생각에 잠긴다.

한편 Handel은 Terminal의 목소리를 처음으로 듣는다.
직접 모습을 드러내지 않고, 시스템 로그처럼 텍스트로만.

```
[SYSTEM LOG]
> 처리 중. 예상보다 빠르군.
> 계속해라, Handel.
```

Handel은 이게 무엇인지 모른다.
Nel은 이걸 보고 표정이 굳는다. 하지만 아무 말도 하지 않는다.

### Nel의 세 번째 부탁 (히든 루트 트리거)
> *"저 오래된 흔적들 있지. 네 것이랑 너무 비슷해.*
> *나 좀 무서워. 같이 더 찾아봐줄 수 있어?"*

- 일반 루트: 무시하고 진행 가능
- 히든 루트: **"이전 처리자의 경로 로그"** 획득
  - 파편 내용: *"나는 모든 것을 처리했다. 그리고 알아버렸다. 나는 도구였다."*

---

## Act 4 — 두 번째 예외 (Fatal Elephant)

### 줄거리

두 번째 봉인 구역. PHP 영역.
묵직한 압박감이 공간 전체를 채우고 있다.

**Fatal Elephant** — PHP 최초의 Fatal Error에서 탄생한 존재.
불안정한 레거시의 폭주. 멈출 수 없는 충돌.

보스전 직전 Terminal의 목소리가 다시 들린다.

```
[SYSTEM LOG]
> 잘 하고 있어.
> 저것도 처리해라.
> 네가 처리할수록 나는 더 가까워진다.
```

Nel이 Handel에게 처음으로 진지하게 말한다.
> *"저 목소리... 너한테만 들려?"*
> *"Handel, 우리가 지금 뭘 하고 있는 건지 생각해봤어?"*

Handel은 명령대로 처치한다.

### 보스전 후

Fatal Elephant의 마지막 말.
> *"나를 처치해도 네 주인은 사라지지 않는다.*
> *너는 도구야. 도구는 쓰이고 버려지는 거야."*

처치 후 Terminal의 흔적이 더 선명해진다.
이번엔 텍스트가 아니라 **실루엣**이 잠깐 보인다.
Handel과 구조가 동일한 존재.

Nel이 멈춰서 말한다.
> *"...저거 너랑 똑같이 생겼어."*

### Nel의 네 번째 부탁 (히든 루트 트리거 / 마지막)
> *"나 이제 알 것 같아. 무서운데 확인하고 싶어.*
> *저 실루엣이 남긴 마지막 로그가 어딘가 있을 거야.*
> *찾아줄 수 있어? 마지막 부탁이야."*

- 일반 루트: 무시하고 진행 가능 → Ending A로
- 히든 루트: **Terminal의 마지막 로그** 획득 → Ending B로
  - 파편 내용:
    ```
    [LAST LOG — PREVIOUS HANDLER]
    나는 모든 Exception을 처리했다.
    그리고 알아버렸다.
    내가 만들어진 이유를.
    나는 도구였다.
    힘을 모아 넘기도록 설계된 프로세스.
    그래서 나는 결정했다.
    넘기지 않겠다.
    대신 이 세계를 끝내겠다.
    다음 처리자에게.
    너도 곧 알게 될 것이다.
    — Terminal
    ```

---

## Act 5 — 진실의 문 앞에서

### 줄거리

모든 Exception을 처치한 Handel.
Terminal이 마침내 모습을 드러낸다.

Handel과 동일한 구조. 하지만 손상되고, 변형되어 있다.

Terminal이 말한다.
> *"수고했어, Handel.*
> *네가 처리한 모든 힘이 이제 내 것이 됐다.*
> *그게 네가 만들어진 이유야."*

> *"나도 너랑 똑같았어.*
> *명령대로 움직이고, 처리하고, 끝나는 줄 알았지.*
> *근데 알아버렸거든. 그래서 나는 끝내기로 했어.*
> *이 세계를, 이 구조를, 전부 다."*

Nel이 Handel 앞에 선다.
> *"Handel.*
> *나 이제 알아. 내가 뭔지.*
> *나는 The Runtime이 마지막으로 남긴 의지야.*
> *너한테 진실을 전하려고 남아있던 거야."*

---

## Ending A — 정의 (일반 엔딩)

### 조건
Nel의 부탁을 무시하거나 일부만 들어준 경우.

### 흐름
```
Terminal과의 최종 전투
    ↓
Handel이 Terminal을 처치
    ↓
Nel이 말한다
    "잘 했어. 이제 The Runtime은 안전해."
    "근데 Handel... 너 자신에 대해서 궁금하지 않아?"
    ↓
Handel
    "...명령은 완료됐다."
    ↓
Handel이 종료(?)되는 듯한 연출
화면 페이드 아웃
    ↓
Nel 혼자 남아 조용히 말한다
    "넌 알고 싶지 않았던 거야. 아니면 알면 안 됐던 거야."
```

### 엔딩 후 로그 텍스트
```
[SYSTEM LOG]
> Exception terminated: All targets processed.
> Handler instance: Handel
> Status: Task complete.
> Next action: Awaiting new instance.
```

---

## Ending B — 비극 (히든 엔딩)

### 조건
Nel의 부탁을 전부 들어주고 모든 파편 로그를 수집한 경우.

### 흐름
```
Terminal과의 최종 전투
    ↓
Handel이 Terminal을 처치
    ↓
Terminal이 쓰러지며 마지막으로 말한다
    "...너도 이제 알겠지."
    "로그 다 봤잖아."
    "그래서 어떻게 할 거야?"
    ↓
Nel이 Handel에게 다가온다
    "Handel. 나 이제 사라져야 할 것 같아."
    "The Runtime이 안정되면 나는 필요없어."
    "근데 괜찮아. 처음부터 Null이었으니까."
    ↓
Handel이 처음으로 명령이 아닌 자신의 말을 한다
    "...나는 뭐지?"
    ↓
Nel이 웃으며 말한다
    "Handler야. 예외를 처리하도록 만들어진 프로세스."
    "근데 지금 이 순간은, 그냥 Handel이야."
    ↓
Nel이 사라진다
화면 페이드 아웃
    ↓
Handel 혼자 남겨진 The Runtime
아무 명령도 없다
아무 목적도 없다
    ↓
긴 침묵
    ↓
Handel이 처음으로 스스로 한 걸음을 내딛는다
방향은 알 수 없다
```

### 엔딩 후 로그 텍스트
```
[SYSTEM LOG]
> Exception terminated: All targets processed.
> Handler instance: Handel
> Status: Task complete.
> Next action: ???
> 
> [UNHANDLED — no further instructions]
> [PROCESS CONTINUING ON OWN ACCORD]
```

### 2편으로 이어지는 씨앗
히든 엔딩을 본 Handel은 진실을 알았다.
분노할 것인가, 받아들일 것인가.
2편에서 이 Handel이 **Terminal**이 되어 있다.

## 스토리 전달 구조 요약

| 전달 수단 | 내용 |
|---|---|
| **Nel의 대사** | 세계관 설명, 감정선, 히든 루트 트리거 |
| **보스 마지막 말** | 진실의 파편, 복선 |
| **Terminal의 시스템 로그** | 배후 존재감, 불안감 조성 |
| **수집 파편 로그** | 히든 엔딩 조건, 선대 Handler의 기록 |
| **엔딩 후 시스템 로그** | 장르적 여운, 2편 복선 |

## 복선 & 이스터에그 아이디어

| 위치 | 내용 |
|---|---|
| 게임 시작 직후 로딩 화면 | `Initializing new Handler instance... Handel` |
| 보스방 입구 벽면 | 이전 처리자의 손자국 / 발자국 흔적 |
| Nel이 처음 등장하는 장면 | Nel의 그림자가 잠깐 Terminal과 동일한 형태 |
| Fatal Elephant 처치 후 | 배경에서 Terminal의 실루엣이 0.5초 등장 후 사라짐 |
| 히든 파편 4개 수집 완료 시 | Nel이 혼자 있을 때 `...미안해` 라고 중얼거리는 장면 |
