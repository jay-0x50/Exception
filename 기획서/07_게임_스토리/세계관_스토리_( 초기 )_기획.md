# 세계관 & 스토리 기획 — Exception

<aside>

**목적:** `Exception`의 세계관 배경, 플레이어 정체, 보스의 의미, 반전 구조를 정의한다.
스토리는 직접 설명하지 않고 **아이템 설명 / 환경 암시 / 보스 처치 후 로그 텍스트**로 파편적으로 전달한다. (엘든링 스타일)

</aside>

---

## 1. 세계관 — The Runtime

태초에 **The Runtime**이 있었다.

모든 언어, 모든 시스템, 모든 프로세스가 태어나는 근원적 실행 환경. 인간의 세계에서 "컴퓨터"라 부르는 것들의 이면에 존재하는 거대한 층위. 이곳에서 코드는 단순한 명령어가 아니라 **살아있는 법칙**이고, 언어는 **신앙**이며, 실행되는 프로세스들은 **생명**이다.

그리고 그 세계를 지배하는 존재들이 있었다 — **Primordial Exceptions**.

인류가 처음 특정 언어를 만들고, 처음 그 언어로 오류를 일으켰을 때 탄생한 원초적 존재들. Python의 첫 번째 오류에서 태어난 것, PHP의 첫 번째 충돌에서 깨어난 것. 이들은 **오류 그 자체가 신격화된 존재**들이다.

수천 번의 버전 업데이트, 수억 번의 디버깅을 거치며 이 존재들은 **The Runtime 깊은 곳에 봉인**되었다. 인류가 코드를 정제할수록 이들은 약해졌고, 결국 잠들었다.

---

## 2. 현재 — 붕괴의 시작

어느 날, **The Runtime이 흔들리기 시작했다.**

원인은 하나 — 누군가 의도적으로 봉인을 건드렸다. 단순한 실수가 아니라, **누군가가 Primordial Exceptions를 깨우려 하고 있다.**

봉인이 풀린 구역부터 The Runtime은 변질되기 시작했다. 정상적으로 실행되던 프로세스들이 오염되고, 법칙이 무너지고, 봉인됐던 존재들이 하나씩 눈을 뜨기 시작한다.

---

## 3. 플레이어 — Handler

플레이어의 이름은 **Handler**.

The Runtime 어딘가에서 갑자기 인스턴스화된 존재. 기억도 없고, 자신이 무엇인지도 모른다. 다만 하나의 명령어만 내장되어 있다 —

```
TASK: Terminate all unhandled exceptions.
```

Handler는 자신이 누군가에 의해 **의도적으로 생성된 프로세스**라는 것을 모른다. 그저 내장된 목적대로 움직일 뿐이다. 보스를 처치할수록 파편적인 로그가 열리고, 자신이 왜 만들어졌는지 서서히 드러난다.

---

## 4. 반전 구조 (엔딩 방향)

보스들을 하나씩 처치하며 열리는 로그 파편들. 그 안에 담긴 진실 —

**Handler를 만든 건 The Runtime을 지키려는 자가 아니었다.**

봉인을 푼 자와, Handler를 만든 자가 **같은 존재**였다. Primordial Exceptions를 깨워서 The Runtime을 혼돈에 빠뜨리고, 그 혼돈 속에서 Handler가 모든 Exception을 처치하게 만든 뒤 — **그 힘을 흡수하려는 계획**이었다.

Handler는 처음부터 도구였다. 그리고 지금, 마지막 Exception을 처치하려는 순간 그 사실을 알게 된다.

> *멈출 것인가, 아니면 도구인 채로 끝낼 것인가.*

---

## 5. 스토리 전달 방식

직접적인 내레이션이나 대사보다 **파편적 암시**로 전달한다.

| 전달 수단 | 내용 예시 |
|---|---|
| 아이템/오브젝트 설명 | 보스방 입구 오브젝트에 붙은 짧은 로그 텍스트 |
| 환경 암시 | 보스방 바닥의 봉인 문양, 벽의 오염된 코드 |
| 보스 처치 후 로그 | 보스가 남긴 마지막 오류 메시지 형식의 텍스트 |
| 체크포인트 근처 파편 | 과거 프로세스들의 잔재, 짧은 독백 로그 |

## 6. 보스 — Primordial Exceptions

각 보스는 단순한 적이 아니라 **봉인에서 깨어난 원초적 오류의 신**이다.

### MVP 보스 — Serpent.py

Python 언어의 첫 번째 오류들에서 탄생한 두 존재. 하나의 언어에서 갈라진 두 개의 예외.

| 보스 | 모티브 | 본질 | 처치 후 로그 |
|---|---|---|---|
| **Serpent.py (골드)** | Python 최초의 `IndentationError` | 질서와 구조를 증오하는 존재. 들여쓰기 하나로 세계가 무너진다는 것을 안다. | *"왜 너는 우리를 지우려 하는가. 우리가 없었다면 너도 없었다."* |
| **Serpent.py (블루)** | Python 최초의 `RecursionError` | 끝없이 자신을 호출하는 존재. 멈추는 법을 잃어버렸다. | *"...너도 결국 반복될 것이다."* |

### 확장 보스 후보

| 보스 | 모티브 | 본질 |
|---|---|---|
| **Fatal Elephant** | PHP 최초의 `Fatal Error` | 묵직한 돌진과 충격파. 불안정한 레거시의 폭주. |
| **Query Dolphin** | MySQL 최초의 `Deadlock` | 데이터 파편과 물결. 서로를 기다리다 영원히 멈춘 존재. |
| **Regex Camel** | Perl 최초의 파국적 정규식 | 변칙 이동, 실타래 패턴. 읽을 수 없는 혼돈 그 자체. |
