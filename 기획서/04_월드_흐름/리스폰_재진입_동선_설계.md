# 리스폰 & 재진입 동선 설계

<aside>

**목적:** 패배 후 재도전까지의 **리스폰 흐름과 동선**을 정의한다.
"죽고 나서 다시 보스에게 돌아오는 과정"이 지루하지 않아야 데모에서 재도전 의지를 유지할 수 있다.

</aside>

---

## 1. 리스폰 흐름 전체 시퀀스

```
[플레이어 HP 0 → 사망]
    ↓
사망 연출 (AM_Player_Death + 페이드 아웃, 약 2~3s)
    ↓
리스폰 처리
    - 플레이어 체크포인트 위치로 이동
    - HP/스태미나 전량 회복
    - 인벤토리/스킬 상태 유지 (데모 기준 변경 없음)
    ↓
보스 상태 초기화
    - 보스 HP/그로기 전량 회복
    - 보스 위치 원위치
    - 보스 AI 비활성화 (재진입 전까지)
    - 페이즈 초기화 (IsPhase2 = false)
    ↓
화면 페이드 인 + 체크포인트에서 플레이어 조작 가능
    ↓
[플레이어 재이동 → 보스방 재도전]
```

---

## 2. 재진입 동선 옵션 (Must 1개 선택)

보스방까지의 재이동이 데모의 리플레이성에 직접 영향을 미침.
아래 세 가지 중 **데모 스코프에 맞게 1개를 Must로 선택**한다.

### 옵션 A: 숏컷 (Shortcut) — 권장

- 보스 최초 처치 후 또는 리스폰 후, 보스방 바로 앞까지 연결되는 **지름길 문/통로 개방**
- 처음 진입 시에는 긴 루트 → 이후 재도전은 짧은 루트
- 소울라이크 장르의 관례적 설계 (다크소울 시리즈 참조)

| 항목 | 내용 |
| --- | --- |
| 숏컷 개방 조건 | 최초 보스방 진입 완료 시 자동 개방 |
| 숏컷 이동 시간 | 15~30초 내 보스방 도달 |
| 구현 방법 | 잠긴 문 Actor → 트리거 이벤트로 `Unlock` 상태 전환 + 문 열림 애니 |

### 옵션 B: 즉시 재도전 선택지 — 간단 구현

- 체크포인트(휴식 포인트)와 상호작용 시 "즉시 보스방 앞으로 이동" 옵션 제공
- 구현: 선택지 UI → 확인 → `SetActorLocation(BossRoomEntrance)`

| 항목 | 내용 |
| --- | --- |
| UI | "다시 도전하시겠습니까?" 선택지 (예/아니오) |
| 이동 방식 | 페이드 아웃 → SetActorLocation → 페이드 인 |
| 단점 | 필드 이동감이 완전히 생략되어 데모 볼륨이 얇게 느껴질 수 있음 |

### 옵션 C: 풀 루트 반복 — 비권장

- 매 재도전마다 필드 전체를 다시 이동
- 필드 길이가 1~3분이므로 데모에서는 리플레이 피로감이 큼
- **데모 목적에 맞지 않으므로 비권장**

## 3. 권장 구성: 옵션 A + B 조합

```
리스폰 → 체크포인트 도착
    ↓
선택지 제공 (체크포인트 상호작용 시)
    ├── [즉시 재도전] → 페이드 아웃/인 후 보스방 앞 스폰
    └── [직접 이동] → 숏컷 통로로 30초 내 이동
```

- 플레이어가 "조작 감각을 다시 잡고 싶으면" 직접 이동 선택
- "빠르게 재도전하고 싶으면" 즉시 재도전 선택
- 양쪽을 모두 제공하면 다양한 플레이 스타일에 대응 가능

## 4. 보스 상태 초기화 규격

리스폰 시 보스는 **완전히 초기 상태로 리셋**되어야 한다.

| 항목 | 초기화 내용 |
| --- | --- |
| HP | MaxHP로 전량 회복 |
| 그로기 게이지 | 0으로 리셋 |
| 페이즈 | `IsPhase2 = false` |
| 위치/회전 | 보스방 스폰 포인트로 복귀 |
| AI | BT 정지 → 재진입 시 재시작 |
| 상태 | `ECharacterState::Idle` |
| Intro 연출 | 재도전 시 스킵 또는 단축 버전 재생 |

```cpp
void ABossRoom::ResetBoss()
{
    if (!IsValid(BossRef)) return;

    // 위치 초기화
    BossRef->SetActorLocationAndRotation(
        BossSpawnTransform.GetLocation(),
        BossSpawnTransform.GetRotation(),
        false, nullptr,
        ETeleportType::TeleportPhysics
    );

    // 스탯 초기화
    BossRef->StatComponent->ResetAllStats();

    // 페이즈 초기화
    BossRef->SetIsPhase2(false);
    BossRef->GetBlackboardComponent()->SetValueAsBool("IsPhase2", false);

    // AI 비활성 (재진입 시 활성화)
    BossRef->GetAIController()->StopBehaviorTree(BossBT);

    // 상태 초기화
    BossRef->SetCharacterState(ECharacterState::Idle);
}
```

## 5. 체크포인트 시스템 규격

| 항목 | 내용 |
| --- | --- |
| 위치 | 필드 시작 지점 (보스방에서 1~3분 거리) |
| 상호작용 | 진입 시 자동 저장 또는 `F` 키로 수동 활성화 |
| 효과 | HP/스태미나 회복 + 리스폰 위치 갱신 |
| 보스 관계 | 체크포인트 활성화 시 보스 HP는 초기화 **하지 않음** (단, 사망 리스폰 시에는 초기화) |
| 연출 | 활성화 파티클/사운드 (선택) |

## 6. 튜닝 체크리스트

- [ ]  리스폰 후 체크포인트에서 플레이어 HP/스태미나가 완전히 회복되는가
- [ ]  보스 HP/그로기/페이즈가 완전히 리셋되는가
- [ ]  즉시 재도전 선택 시 보스방 Intro가 스킵 또는 단축 재생되는가
- [ ]  숏컷 통로가 최초 진입 후 정확히 개방되는가
- [ ]  재진입 후 보스 AI가 즉시 활성화되어 전투가 시작되는가
- [ ]  리스폰 → 재도전까지 총 소요 시간이 30초 이내인가 (즉시 재도전 기준)
