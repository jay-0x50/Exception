# 플레이어 피격 리액션 상세

<aside>

**목적:** "맞는 느낌"을 만드는 **플레이어 피격 리액션 규격**을 정의한다.
리액션 타입별 경직/넉백/에어본/사망 처리와 예외 규칙을 명시하여,
판정 구현 시 빠짐없이 대응할 수 있도록 한다.

</aside>

---

## 1. 리액션 타입 정의

| ReactionType | 발생 조건 | 설명 |
| --- | --- | --- |
| `Light_Hit` | DMG_Boss_Normal | 짧은 경직. 이동/공격 잠깐 끊김. |
| `Heavy_Hit` | DMG_Boss_Smash (근거리) | 강한 경직 + 뒤로 밀림(선형 넉백). |
| `Knockback` | DMG_Boss_Smash (직격) | 공중으로 튀어오름 → 낙하 → 바닥 착지. |
| `Death` | HP ≤ 0 | 사망 연출 후 리스폰 처리. |

---

## 2. 리액션별 상세 규격

### 2-1) Light_Hit (경량 경직)

- **몽타주:** `AM_Player_HitLight`
- **경직 시간:** `0.25s`
- **이동:** 경직 중 이동 입력 차단
- **무적 여부:** 없음 (경직 중 추가 피격 가능)
- **히트스톱:** `0.08s` (공격자/피격자 동시 정지)
- **카메라 쉐이크:** `CS_LightHit`
- **추가 피격 처리:** 경직 중 피격 시 경직 리셋 (새 리액션으로 덮어쓰기)

### 2-2) Heavy_Hit (강 경직 + 밀림)

- **몽타주:** `AM_Player_HitHeavy`
- **경직 시간:** `0.5s`
- **넉백 거리:** 보스 방향 반대로 `200` 유닛
- **넉백 처리:** `LaunchCharacter()` 수평 방향만 (`bXYOverride=true, bZOverride=false`)
- **무적 여부:** 넉백 이동 중 `0.1s` 무적 (연속 피격 방지용)
- **히트스톱:** `0.15s`
- **카메라 쉐이크:** `CS_HeavyHit`
- **이동:** 넉백 중 이동 입력 차단 (착지 후 회복)

### 2-3) Knockback (에어본)

- **몽타주:** `AM_Player_Knockback`
- **발사 벡터:** 보스 방향 반대 X/Y + Z 방향으로 `LaunchCharacter(FVector(X, Y, 500))`
- **공중 상태:** `IsInAir()` 동안 입력 차단
- **착지 판정:** `OnLanded()` 이벤트로 감지
- **착지 후 처리:**
    - 착지 몽타주(`AM_Player_LandHard`) 재생 후 Idle 복귀
    - 착지 순간 `CS_Explosion` 카메라 쉐이크
- **에어본 중 추가 피격:**
    - 에어본 중 피격은 **무시** (에어 콤보 방지)
    - 단, `Knockback` 타입 공격은 예외로 히트 허용 가능 (선택)
- **무적 여부:** 에어본 전 구간 무적

### 2-4) Death (사망)

- **발동 조건:** `StatComponent`에서 `CurrentHP <= 0` 감지 시 Delegate 발동
- **몽타주:** `AM_Player_Death`
    - 쓰러지는 방향: 보스 방향 기준 뒤로 도미노
- **처리 순서:**
    1. 상태 → `ECharacterState::Dead`
    2. 입력 전체 차단 (`DisableInput`)
    3. 사망 몽타주 재생
    4. 몽타주 종료 후 `0.5s` 딜레이
    5. 화면 페이드 아웃 (블랙)
    6. 리스폰 로직 호출
- **사망 중 추가 피격:** 모든 피격 무시 (`Dead` 상태 체크로 TakeDamage 조기 리턴)
- **히트스톱:** 없음 (사망은 히트스톱 생략, 연출 집중)
- **사운드:** 사망 SFX + (선택) 슬로우 모션 0.3배속 0.5s

## 3. 상태별 피격 가능 여부 매트릭스

| 플레이어 상태 | Light_Hit | Heavy_Hit | Knockback | 비고 |
| --- | --- | --- | --- | --- |
| Idle | ✅ | ✅ | ✅ | 정상 피격 |
| Attack | ✅ | ✅ | ✅ | 공격 중단 후 경직 |
| Dodge (무적 구간) | ❌ | ❌ | ❌ | i-frame으로 전체 무시 |
| Parry (판정 구간) | ❌ Normal만 | ✅ | ✅ | 패링 판정 구간엔 Normal 흡수 |
| HitReaction (Light) | ✅ (덮어쓰기) | ✅ (덮어쓰기) | ✅ | 경직 중 더 강한 리액션으로 교체 |
| Knockback (에어본) | ❌ | ❌ | 선택 | 에어 콤보 방지 |
| Dead | ❌ | ❌ | ❌ | 전체 무시 |

## 4. 피격 처리 흐름 (코드 관점)

```
보스 무기 히트 (AnimNotifyState Overlap)
→ IExCombatInterface::OnWeaponHit(AtkType, DamageData)
→ AExPlayerCharacter::TakeDamage()
    → 상태 체크 (Dead/Invincible이면 조기 리턴)
    → StatComponent::ApplyDamage(DamageValue)
    → ApplyHitReaction(ReactionType)
        → 몽타주 선택 및 재생
        → LaunchCharacter (Knockback/Heavy_Hit)
        → HitStop 타이머 세팅
        → CameraShake 발동
```

## 5. HitStop (히트스톱) 처리 규격

히트스톱은 "때리는 맛"과 "맞는 무게감" 모두에 영향을 주는 핵심 요소.

| 구현 방식 | 내용 |
| --- | --- |
| 방법 | `CustomTimeDilation`을 보스/플레이어 모두에 적용 |
| 히트스톱 중 값 | `TimeDilation = 0.05` (거의 정지) |
| 히트스톱 종료 | `FTimerHandle`로 지정 시간 후 `TimeDilation = 1.0` 복구 |
| 적용 대상 | 공격자(보스) + 피격자(플레이어) 동시 적용 |
| 카메라 | 히트스톱 중에도 카메라 TimeDilation은 1.0 유지 (카메라가 멈추면 이질감) |

## 6. 튜닝 체크리스트

- [ ]  Light_Hit 경직이 너무 길어 조작 답답함을 주지 않는가
- [ ]  Heavy_Hit 넉백 거리가 보스 공격 범위 밖으로 안전하게 밀어주는가
- [ ]  Knockback 에어본 높이/거리가 지형 밖으로 날아가지 않는가
- [ ]  경직 중 연속 피격 시 리액션 덮어쓰기가 자연스러운가
- [ ]  히트스톱 길이가 너무 길어 리듬감을 해치지 않는가
- [ ]  사망 연출 → 페이드 → 리스폰 전환이 매끄럽게 연결되는가
