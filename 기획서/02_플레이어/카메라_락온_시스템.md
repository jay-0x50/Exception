# 카메라 & 락온 시스템

<aside>

**목적:** 소울라이크 전투의 핵심 UX인 **락온(Lock-on) 카메라 시스템**의 동작 규격과 구현 방향을 정의한다.
카메라는 단순 시점 도구가 아니라 **전투 판단력과 직결되는 게임플레이 요소**임을 전제로 설계한다.

</aside>

---

## 1. 락온 시스템 개요

### 1) 락온 여부 결정

- 본 데모는 **락온 있음** 방식으로 구현
- 보스가 1체이므로 **자동 락온** (범위 내 진입 시 자동 타겟 지정 또는 수동 키 1회)
- 락온 키: `Tab` 또는 `마우스 우클릭 홀드` (프로젝트 컨벤션에 맞게 조정)

### 2) 락온 활성화 조건

| 조건 | 내용 |
| --- | --- |
| 거리 | 플레이어 ↔ 보스 거리 ≤ `LockOnRange` (기본값: 1500) |
| 시야 | LineTrace로 LoS 확인 (벽 뒤 락온 방지) |
| 보스 상태 | 보스가 `Dead` 상태가 아닐 것 |
| 플레이어 상태 | 플레이어가 `Dead` 상태가 아닐 것 |

### 3) 락온 해제 조건

- 수동 해제: 락온 키 재입력
- 자동 해제 (아래 중 하나 성립 시):
    - 보스와의 거리가 `LockOnBreakRange` (기본값: 2000) 초과
    - 보스 사망
    - 플레이어 사망

## 2. 카메라 동작 규격

### 2-1) 락온 OFF (자유 카메라)

- 기본 SpringArm 카메라
- ArmLength: `400`
- 마우스/우스틱으로 자유 회전

### 2-2) 락온 ON

| 항목 | 수치 (기본값) | 비고 |
| --- | --- | --- |
| ArmLength | 350 | 자유 카메라보다 약간 가깝게 |
| 카메라 높이 오프셋 | +80 | 보스를 약간 위에서 내려다보는 구도 |
| 수평 오프셋 | +60 (우측) | 플레이어가 화면 좌측에 위치, 보스 우측 배치 |
| 타겟 포커스 포인트 | 보스 루트 + Z오프셋 120 | 보스 상체 중심을 바라보도록 |
| 회전 보간 속도 | 8.0 | 너무 빠르면 멀미, 너무 느리면 반응 지연 |
| LoS 막힘 시 카메라 거리 | 자동 단축 (최소 150) | SpringArm의 기본 충돌 처리 활용 |

### 2-3) 카메라 충돌 처리

- `USpringArmComponent`의 `bDoCollisionTest = true` 사용
- 벽/기둥 뒤로 카메라가 들어갈 경우 ArmLength 자동 단축
- 카메라가 플레이어 캐릭터를 **항상 화면에 유지**하는 것을 최우선으로

---

## 3. 락온 중 이동 방향 처리

락온 중에는 캐릭터가 보스를 향해 자동으로 정면을 유지해야 함.

### 이동 입력 처리

- 입력 방향 = 카메라 기준 상대 방향
- `WASD / 좌스틱` → 보스 중심 기준 8방향 이동
    - W: 보스 방향으로 전진
    - S: 보스 방향에서 후퇴
    - A/D: 보스 기준 좌/우 스트레이프

### 캐릭터 회전

- 락온 중에는 `bUseControllerRotationYaw = false`
- 대신 매 프레임 보스 방향으로 `FMath::RInterpTo`로 부드럽게 회전
- 회전 속도(InterpSpeed): `10.0` (기본값, 튜닝 가능)

## 4. C++ 구현 가이드

### 4-1) 추천 컴포넌트 구조

```cpp
// ExPlayerCharacter.h

UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
TObjectPtr<USpringArmComponent> CameraBoom;

UPROPERTY(VisibleAnywhere, BlueprintReadOnly)
TObjectPtr<UCameraComponent> FollowCamera;

// 락온 타겟
UPROPERTY(BlueprintReadOnly)
TObjectPtr<AActor> LockOnTarget;

bool bIsLockedOn = false;

UFUNCTION(BlueprintCallable)
void ToggleLockOn();

UFUNCTION(BlueprintCallable)
AActor* FindLockOnTarget(float SearchRadius) const;

void UpdateLockOnCamera(float DeltaTime);
```

### 4-2) Tick 처리

```cpp
void AExPlayerCharacter::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    if (bIsLockedOn && IsValid(LockOnTarget))
    {
        UpdateLockOnCamera(DeltaTime);

        // 자동 해제 거리 체크
        float Dist = FVector::Dist(GetActorLocation(), LockOnTarget->GetActorLocation());
        if (Dist > LockOnBreakRange)
        {
            ToggleLockOn(); // 자동 해제
        }
    }
}
```

> **주의:** Tick 안에 있지만 락온 상태일 때만 실행되므로 오버헤드 최소화.
> 락온 비활성 시 카메라 업데이트 생략.

## 5. 튜닝 체크리스트

- [ ]  락온 전환 시 카메라 팝핑 없이 부드럽게 전환되는가
- [ ]  보스가 빠르게 이동할 때 카메라가 따라가는 속도가 적절한가
- [ ]  보스 돌진(P2-1 ShadowDash) 중 카메라가 플레이어를 잃지 않는가
- [ ]  벽 근처에서 카메라가 캐릭터를 가리지 않는가
- [ ]  락온 해제 후 자유 카메라로 자연스럽게 복귀되는가

---

## 6. MVP 기준

- **Must:** 락온 ON/OFF 토글 + 락온 중 보스 추적 카메라 + 자동 해제
- **Nice:** 카메라 흔들림(히트스톱 연동) 강도 락온 여부에 따라 차등 적용
